Sprint 0

API

Bij API werkte ik aan een Astro-project rondom een Harry Potter spell-book. Met verschillende Web API's zoals SpeechSynthesis en Web Transitions.

    Belangrijkste inzichten

  • - Hoe je Dynamische pagina's kan genereren met getStaticPaths() in Astro.
  • - Hoe je Data kan ophalen en verwerken vanuit een externe API.
  • - Het koppelen van spraakherkenning koppelen aan navigatie.
  • - Web transitions kunnen een website veel meer interactie geven.
Het resultaat is een website waar gebruikers door een spellbook kunnen bladeren, met hun stem.

Browser Tech

Tijdens Browser Technologies bouwde ik een formulier voor aangifte erfbelasting in de stijl van de NS. Hier lag de focus op formulieren, progressive enhancement en validatie.

    Belangrijkste inzichten

  • - Goede formulieren beginnen bij semantische HTML (fieldset, legend, label).
  • - Mobile-first ontwerpen werkt efficiënter.
  • - Progressive enhancement maakt websites robuuster.
  • - Validatie moet samenwerken met dynamische patterns.
De uitkomst is een volledig functioneel en toegankelijk formulier dat werkt mét en zonder JavaScript, inclusief dynamische velden, validatie en foutmeldingen. UX/UI draait niet alleen om uiterlijk, maar vooral om duidelijke functionaliteit.

Bekijk project

CSS

Bij CSS werkte ik aan een creatieve Artifaction-opdracht waarbij ik experimenteerde met grids, shapes, selectors en random layouts.

    Belangrijkste inzichten

  • - De :has() selector opent veel nieuwe mogelijkheden in CSS.
  • - Moderne CSS kan veel logica oplossen zonder JavaScript.
  • - CSS Grid draait om verhoudingen en structuur, niet alleen pixels.
  • - Randomness in CSS vraagt om creatieve oplossingen.
  • - Nieuwe CSS-functies zoals random() kunnen krachtig zijn, maar zijn lastig te beheersen.
De uitkomst is een experimentele generatieve grid-layout met patronen, pseudo-random plaatsing, dark/light mode en interactieve CSS-effecten

Bekijk project

HCD

Voor HCD ontwikkelde ik een Chrome extension waarmee gebruikers zelf de focus-startpositie van websites kunnen bepalen, gericht op toegankelijkheid en screenreadergebruikers.

    Belangrijkste inzichten

  • - Ontwerpen voor één specifieke gebruiker levert vaak betere oplossingen op voor meerdere mensen.
  • - Accessibility draait om kleine gebruikservaringen zoals focus, geluid en navigatie.
  • - Testen met echte gebruikers levert de meest waardevolle feedback op.
  • - Audio feedback moet ondersteunend zijn en niet storend naast screenreaders.
De uitkomst is een werkende Chrome extension waarmee focusposities opgeslagen en hersteld kunnen worden, inclusief shortcuts, focusbeheer en audio cues voor toegankelijkheid.

Peter Paul Koch Weekly Nerd 1

Wat is een browser?

Een browser bestaat uit:

  • HTML
  • CSS
  • DOM (Document Object Model) - Kopie van HTML geschikt voor JavaScript.
  • AOM (Accessibility Object Model) - DOM met functies voor screenreaders.
  • Interface
  • JavaScript engine

Parser

Een parser is een softwarecomponent dat invoer ontleedt en omzet in een gestructureerd format.

Rendering engine

De rendering engine is verantwoordelijk voor parsing en rendering van HTML en CSS, en voor de constructie van DOM- en AOM-trees.

De rendering engine is niet verantwoordelijk voor de interface of de JavaScript engine.

JavaScript en DOM

  • Core is wat JavaScript een programmeertaal maakt.
  • DOM is de manier waarop JavaScript HTML kan aanspreken en wijzigen.

Render Blocking

CSS en JavaScript blokkeren rendering, tenzij JavaScript async of defer gebruikt.

  • Do it now: script met src
  • Do it later: script met defer
  • Do it whenever: script met async

Backwards Compatibility

Principe: als iets in een browser heeft gewerkt, moet het altijd in die browser blijven werken.

zorgt ervoor dat de browser de moderne rendering engine gebruikt.

Geschiedenis van browsers

WWW Browser (1980)

Allereerste browser, gebouwd bij CERN door Tim Berners-Lee en consorten.

Mosaic (1993)

Radicale innovatie. Introduceerde de img-tag.

Netscape (1994)

Eerste echte browser. Commercieel en betaald.

Internet Explorer (1995)

Gratis browser van Microsoft.

Opera (1995)

Kleinere browser.

Internet Explorer 5 Mac (2000)

Anders dan Internet Explorer voor Windows.

Konqueror (2000)

Omzeilde veel problemen en werd later de basis voor Safari.

  • Safari verscheen in 2003.
  • WebKit is de rendering engine van Safari.

Firefox (2004)

Opvolger van Netscape 4 en veel beter.

Safari iOS (2007)

Eerste telefoonbrowser.

Chrome (2008)

Gebruikt tegenwoordig de Blink rendering engine.

Flow (2020)

Eigen rendering engine. Richtte zich voornamelijk op tv-browsers.

Browsers anno 2026

  • Chrome
  • Safari
  • Firefox

Igalia

Spaans opensourcebedrijf met veel browserkennis, maar geen browsermaker.
