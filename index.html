Hackathon

De Hackathon was een samenwerking met SRON, een ruimtevaartbedrijf door studenten. Wij hebben de website ontwikkeld in korte tijd. Mijn inbreng was het 3D modelleren van de satteliet in Blender die gebruikt werd om de verschillende onderdelen van de satteliet weer te geven. Verder heb ik de sterrenhemel in de achtergrond gemaakt en ervoor gezorgd dat deze beweegt wanneer je muis beweegt (parralax) Een nieuw inzicht van mij was dat coderen als groep prima kan, hier had ik twijfels over. Je kan ook veel sneller bij een eindresultaat komen.

Sprint 0

Tijdens Sprint 0 heb ik gewerkt aan een website waarop ik mezelf voorstel. Het project begon heel ambitieus met een idee van een 3D geanimeerd badeendje, waarop gebruikers konden klikken om meer informatie over mij te krijgen. Dit was technisch nog te hoog gegrepen. Daarom heb ik mijn aanpak aangepast en ben ik eerst gaan focussen op de minimale eisen van de opdracht. Inzichten die ik tijdens de Sprint-0 heb opgedaan zijn o.a. dat het soms nodig is om een creatief en ambitieus idee iets haalbaarder te maken, zodat ik een werkend eindproduct kan opleveren. Daarom dus eerst de basis, dan extra functies. Zelf proberen helpt beter bij leren. Het is beter om zelf te experimenteren. Dat helpt met kennis onthouden. Werken met Blender en animaties geeft veel mogelijkheden maar kost veel tijd en technische kennis.

API

Bij API werkte ik aan een Astro-project rondom een Harry Potter spell-book. Met verschillende Web API’s zoals SpeechSynthesis en Web Transitions. Belangrijkste inzichten Dynamische pagina’s genereren met getStaticPaths() in Astro. Data ophalen en verwerken vanuit een externe API. Spraakherkenning koppelen aan navigatie. Toegankelijkheid draait om voorspelbaarheid, schaalbaarheid en gebruiksgemak. Web transitions kunnen een website veel meer gevoel en interactie geven. Het resultaat is een website waar gebruikers door een spellbook kunnen bladeren, met hun stem.

Browser Tech

Tijdens Browser Technologies bouwde ik een formulier voor aangifte erfbelasting in de stijl van de NS. Hier lag de focus op formulieren, progressive enhancement en validatie. Belangrijkste inzichten Goede formulieren beginnen bij semantische HTML (fieldset, legend, label). Mobile-first ontwerpen werkt efficiënter. Progressive enhancement maakt websites robuuster. Validatie moet samenwerken met dynamische . De uitkomst is een volledig functioneel en toegankelijk formulier dat werkt mét en zonder JavaScript, inclusief dynamische velden, validatie en foutmeldingen. UX/UI draait niet alleen om uiterlijk, maar vooral om duidelijke functionaliteit.

CSS

Bij CSS werkte ik aan een creatieve Artifaction-opdracht waarbij ik experimenteerde met grids, shapes, selectors en pseudo-random layouts. Belangrijkste inzichten De :has() selector opent veel nieuwe mogelijkheden in CSS. Moderne CSS kan veel logica oplossen zonder JavaScript. CSS Grid draait om verhoudingen en structuur, niet alleen pixels. Randomness in CSS vraagt om creatieve oplossingen. Nieuwe CSS-functies zoals random() kunnen krachtig zijn, maar zijn lastig te beheersen. De uitkomst is een experimentele generatieve grid-layout met patronen, pseudo-random plaatsing, dark/light mode en interactieve CSS-effecten

HCD

Voor HCD ontwikkelde ik een Chrome extension waarmee gebruikers zelf de focus-startpositie van websites kunnen bepalen, gericht op toegankelijkheid en screenreadergebruikers. Belangrijkste inzichten Ontwerpen voor één specifieke gebruiker levert vaak betere oplossingen op. Accessibility draait om kleine gebruikservaringen zoals focus, geluid en navigatie. .js en popup.js communiceren via messages. Testen met echte gebruikers levert de meest waardevolle feedback op. Audio feedback moet ondersteunend zijn en niet storend naast screenreaders. De uitkomst is een werkende Chrome extension waarmee focusposities opgeslagen en hersteld kunnen worden, inclusief shortcuts, focusbeheer en audio cues voor toegankelijkheid.

Meesterproef

Doelen voor de meesterproef: Proberen goed in groepsverband te werken, opdrachten goed verdelen. Iedereen een kans geven om uitdaging op te zoeken. Een duidelijkere rol hebben binnen de groepsopdracht. JS verder onder controle krijgen en meer nieuwe dingen testen. Meer met 3D werken als het kan in de opdracht. Ik vond de minor tot nu toe leerzaam en uitdagend. Ik heb veel vaardigheden ontwikkeld, en ook geleerd hoe belangrijk toegankelijkheid en gebruiksvriendelijkheid is bij het ontwikkelen van websites. Ik heb beter leren inschatten in wat haalbaar is in een bepaalde tijd, en hoe ik een idee eerst in de basis af kan maken en daarna extra functies kan toevoegen. Ik vind het leuk om visueel interessante dingen te maken, zoals 3D modellen en animaties. Dit heb ik ook kunnen toepassen in web development.
